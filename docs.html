⚙️ 功能特性

OpenClaw 是一个个人 AI 助手框架，运行在你的本地机器（或服务器）上，成为消息应用与 AI 之间的桥梁。

适合谁使用？

开发者和高级用户，希望拥有一个随时随地可以消息联系、同时不放弃数据控制权的个人 AI 助手。

核心亮点

  • ✅ 在你自己的设备上运行（Mac / Windows / Linux）
  • ✅ 支持任意 LLM（Anthropic、OpenAI、本地模型等）
  • ✅ 通过主流聊天 App 交互（WhatsApp、Telegram、Discord 等）
  • ✅ 跨会话持久记忆
  • ✅ 浏览器自动化（填表单、爬数据）
  • ✅ 完整系统访问（读写文件、执行脚本）
  • ✅ 可扩展技能平台（社区 + 自建）
  • ✅ 开源，MIT 许可，社区驱动

🚀 快速开始

系统要求

安装前请确认：

  • Node.js 22 或更新版本（推荐 Node 24）
  • macOS、Windows（WSL2 推荐）或 Linux
  • 选择的 AI 模型的 API 密钥
# 检查 Node 版本
node --version

方法一：一键安装脚本（推荐）

自动处理 Node 检测、安装和引导配置：

# macOS / Linux
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows（PowerShell）
iwr -useb https://openclaw.ai/install.ps1 | iex

方法二：npm 全局安装

npm i -g openclaw
openclaw onboard

方法三：从源码构建（开发者推荐）

git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install
pnpm run build
pnpm run openclaw onboard

引导向导步骤

  • 1
    安装：使用上方任一方式完成安装
  • 2
    运行向导：openclaw onboard --install-daemon — 配置认证、Gateway 和频道
  • 3
    检查状态：openclaw gateway status
  • 4
    打开控制台：openclaw dashboard — 在浏览器中直接聊天，无需额外配置
💡 提示： 最快上手方式是直接打开控制台（openclaw dashboard），无需配置任何频道即可立即开始聊天。访问 http://127.0.0.1:18789/。

🤖 Agent 运行时

OpenClaw 运行单一嵌入式 Agent 运行时，使用单一工作区目录作为所有工具的工作目录。

工作区文件

工作区内有几个用户可编辑的关键文件（默认位于 ~/.openclaw/workspace）：

文件用途
AGENTS.md操作指令 + "记忆"，Agent 行为的核心定义
TOOLS.md工具使用笔记和约定（用户维护）
BOOTSTRAP.md首次运行仪式（完成后删除）
USER.md用户画像 + 偏好称谓
注意： 每个新会话的第一轮对话，OpenClaw 会将以上文件内容直接注入 Agent 上下文。空文件会被跳过；过大的文件会被截断。

🧠 记忆与会话

OpenClaw 的记忆系统通过工作区文件实现跨会话持久化。会话历史存储在：

~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl

备份与迁移

将 OpenClaw 迁移到新机器只需三步：

# 1. 在新机器上安装
curl -fsSL https://openclaw.ai/install.sh | bash

# 2. 从旧机器复制状态目录
scp -r ~/.openclaw user@newmachine:~/

# 3. 运行诊断修复
openclaw doctor

🔌 技能与插件

技能系统是 OpenClaw 的超级能力。每个技能是一个包含 SKILL.md 文件的目录，定义元数据和使用说明。

技能加载位置（优先级由高到低）

  • ~/.openclaw/workspace/skills/ — 用户自定义技能
  • ~/.openclaw/skills/ — 安装的社区技能
  • 内置技能 — OpenClaw 自带

从 ClawHub 安装技能

# 列出可用技能
openclaw skills list

# 安装技能
openclaw skills install <skill-name>

# 查看技能信息
openclaw skills info <skill-name>
社区技能市场： 访问 clawhub.ai 下载社区构建的技能——包括 Todoist 自动化、Spotify 控制、健康数据追踪等。

🔧 Gateway 运维

Gateway 是 OpenClaw 的核心进程，管理所有会话、路由和频道连接。

常用命令

# 查看状态
openclaw gateway status
openclaw gateway status --deep

# 服务管理
openclaw gateway install    # 安装为系统服务
openclaw gateway start
openclaw gateway restart
openclaw gateway stop

# 诊断
openclaw doctor             # 审计并修复配置
openclaw logs --follow      # 实时查看日志

多实例运行

# 实例 A
OPENCLAW_CONFIG_PATH=~/.openclaw/a.json \
OPENCLAW_STATE_DIR=~/.openclaw-a \
openclaw gateway --port 19001

# 实例 B
OPENCLAW_CONFIG_PATH=~/.openclaw/b.json \
OPENCLAW_STATE_DIR=~/.openclaw-b \
openclaw gateway --port 19002

❓ 常见问题

OpenClaw 需要付费吗？

OpenClaw 本身完全免费开源（MIT 许可）。费用来自所选 AI 模型的 API 调用费用（自带 API Key）。

支持 Windows 吗？

支持，强烈推荐在 WSL2 下运行以获得最佳兼容性。原生 Windows 也可通过 PowerShell 安装脚本安装。

最低硬件要求是什么？

  • 绝对最低：1 vCPU，1GB RAM
  • 推荐：2GB RAM 或更多（特别是运行多个频道、浏览器自动化或媒体工具时）

数据安全吗？

所有数据存储在你自己的设备上。OpenClaw 不会将你的对话数据上传到任何第三方服务器。AI 模型的 API 调用根据所选提供商的隐私政策处理。

遇到 Anthropic 429 速率限制怎么办？

如果使用 Claude 订阅（setup-token），等待窗口重置或升级计划。如果使用 API Key，在 Anthropic Console 检查用量并提高限额。建议配置备用模型以确保不中断。

如何将 OpenClaw 迁移到新机器？

# 在新机器安装 OpenClaw
curl -fsSL https://openclaw.ai/install.sh | bash

# 从旧机器复制状态
cp -r ~/.openclaw /path/to/backup

# 在新机器恢复并修复
cp -r /path/to/backup ~/.openclaw
openclaw doctor

快速故障排查

  1. 检查 OS 更新和 Node 版本
  2. 确认 Gateway 可达：openclaw gateway status
  3. 检查 Agent/会话状态
  4. 验证 AI 提供商配置和 API Key
  5. 查看日志：openclaw logs --follow
  6. 运行诊断：openclaw doctor
